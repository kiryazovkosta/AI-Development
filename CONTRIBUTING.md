### Code Quality Standards

- **Documentation**: All code should be well-documented
- **README**: Each project must have a comprehensive README.md
- **Dependencies**: Clearly specify all dependencies
- **AI Generation**: Indicate if code was AI-generated and which AI tool was used
- **Testing**: Include tests where applicable

### Project README Requirements

Every project should have a README.md with:
- Project title and description
- AI tool used for generation (e.g., "Generated with GitHub Copilot")
- Prerequisites and dependencies
- Installation instructions
- Usage examples
- Known limitations
- License information (if applicable)

### Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new calculator project to learning-projects`
- `docs: Update PROJECT_CATALOG with task manager app`
- `fix: Correct dependency issues in weather-app`

### Pull Requests

- Keep changes focused and atomic
- Update documentation as needed
- Test your changes before submitting
- Reference related issues if applicable
